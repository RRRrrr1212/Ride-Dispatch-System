# 📱 UI/UX 設計規範文件

> **版本**: v2.0  
> **最後更新**: 2025-12-27  
> **專案**: Ride-Dispatch-System

---

## 📋 總覽

本文件定義 Uber 叫車系統三個客戶端應用程式（乘客端、司機端、管理後台）的 UI/UX 設計規範，目標是打造**現代化、專業、類似真實 Uber App 的使用者體驗**。

### 設計原則

| 原則 | 說明 |
|------|------|
| 🎯 **手機優先** | 乘客端和司機端模擬手機 App 體驗，採用 400x780 尺寸 |
| 🌑 **深色主題** | 使用 Uber 風格的深色配色，減少視覺疲勞 |
| ✨ **流暢動畫** | 添加微動效提升互動體驗 |
| 📱 **可滑動** | 所有頁面支援滑動瀏覽 |
| 🎨 **現代設計** | 圓角卡片、漸層按鈕、優雅間距 |

```

```

```
xl:   32px  - 區塊間距

```

### 圓角系統 (Border Radius)

```

xl:     24px   - 懸浮按鈕、彈窗
full:   9999px - 圓形（頭像）
```

## 🎭 動畫與轉場

### 轉場動畫

| 轉場類型 | 動畫效果 | 時長 |
|---------|---------|------|
| 頁面進入 | Fade In + Slide Up | 300ms |
| 頁面離開 | Fade Out | 200ms |
| 按鈕點擊 | Scale Down → Scale Up | 100ms |
| 卡片選中 | Border Glow + Scale | 200ms |
| 狀態變更 | Color Transition | 400ms |

### 微動效

```java
// 脈動動畫（等待中狀態）
ScaleTransition pulse = new ScaleTransition(Duration.millis(800), node);
pulse.setFromX(1.0);
pulse.setFromY(1.0);
pulse.setToX(1.05);
pulse.setToY(1.05);
pulse.setCycleCount(Timeline.INDEFINITE);
pulse.setAutoReverse(true);

// 進度條動畫
ProgressIndicator progress = new ProgressIndicator();
progress.setStyle("-fx-progress-color: #06C167;");
```
